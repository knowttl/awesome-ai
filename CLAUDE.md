# CLAUDE.md — skills-registry

## What This Project Is

A **skills-registry**: CLI tool + content monorepo for managing reusable AI coding skills, agents, and instructions. Installs content into project-local directories for 8 AI coding assistants (Claude Code, GitHub Copilot, Cursor, Cline, OpenCode, Codex, Windsurf, Roo Code).

## Architecture

- **Zero dependencies**: Pure Bash (`.sh`) + PowerShell (`.ps1`). No Node, Python, or package managers.
- **Dual implementation**: Every script has both `.sh` and `.ps1` versions with identical behavior.
- **Dispatcher pattern**: `bin/skill` routes to `bin/commands/<cmd>.sh`; shared functions in `bin/lib/`.

## File Map

```
bin/skill, bin/skill.ps1          # CLI entry points (dispatchers)
bin/commands/*.sh / *.ps1          # Commands: list, search, info, install, uninstall, sync
bin/lib/*.sh / *.ps1               # Libraries: common, agents, git, lock, profile
skills/<name>/manifest.yaml        # Skill definitions + SKILL.md content files
instructions/<name>/manifest.yaml  # Instruction definitions + content files
profiles/<name>.yaml               # Named bundles that install multiple items
tests/test-*.sh                    # Bash test suites (51 tests, 6 files)
registry.json                      # Generated index — regenerated by bin/skill sync
```

## Conventions

- **YAML parsing**: No `yq`. Uses `awk`/`sed` via `yaml_read_field` and `yaml_read_list` in `common.sh`.
- **JSON handling**: Bash uses `awk`/`sed` template-based string building (no `jq`). PowerShell uses `ConvertTo-Json`/`ConvertFrom-Json`.
- **Color output**: `$'\033[...]'` ANSI codes in `common.sh`, disabled when stdout is not a TTY.
- **Error handling**: `set -euo pipefail` in all Bash scripts. `$ErrorActionPreference = "Stop"` in PowerShell.
- **Naming**: Bash uses `snake_case`. PowerShell uses `Verb-Noun`.

## Common Commands

```bash
bash tests/run-tests.sh          # Run all 51 tests
bin/skill list                   # List all registry items
bin/skill sync                   # Regenerate registry.json from manifests
bin/skill install <name>         # Install a skill/instruction
bin/skill uninstall <name>       # Remove an installed skill/instruction
```

## Rules When Editing

- Always maintain both `.sh` and `.ps1` versions of any script you change.
- After modifying content under `skills/`, `instructions/`, or `profiles/`, run `bin/skill sync`.
- Run `bash tests/run-tests.sh` to verify changes.
- Keep scripts zero-dependency — no `jq`, `yq`, `node`, or other external tools.

## Content Format

Each skill/instruction has a `manifest.yaml`:

```yaml
name: my-skill
type: skill
description: Brief description.
tags: [tag1, tag2]
targets: [claude-code, github-copilot]
files: [SKILL.md]
version: "1.0.0"
```
