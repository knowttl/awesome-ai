# AGENTS.md ‚Äî skills-registry

## What This Project Is

A **skills-registry**: CLI tool + content monorepo for managing reusable AI coding skills, agents, and instructions. Installs content into project-local directories for 8 AI coding assistants (Claude Code, GitHub Copilot, Cursor, Cline, OpenCode, Codex, Windsurf, Roo Code).

## Agent Rules

Rules that govern how the agent works on this project. **Self-updating**: when the user establishes a new behavioral rule (e.g. "always test after X", "from now on, when Y happens do Z", "never Q unless R"), add it as a bullet point below. Follow the format of existing rules. Do not ask for confirmation ‚Äî just add it.

- Always maintain both `.sh` and `.ps1` versions of any script you change.
- After modifying content under `skills/`, `instructions/`, or `profiles/`, run `bin/skill sync`.
- Run `bash tests/run-tests.sh` to verify changes.
- Keep scripts zero-dependency ‚Äî NEVER use `jq`, `yq`, `node`, or other external tools.
- When making structural changes (new naming conventions, new manifest fields, new commands, changed directory layouts), always update `AGENTS.md` to reflect the change.
- Use `snake_case` for Bash functions; `Verb-Noun` cmdlet style for PowerShell.

## File Map

```

registry.json                      # Generated index ‚Äî regenerated by bin/skill sync

```

## Conventions

- **YAML parsing**: NEVER use `yq`. Use `awk`/`sed` via `yaml_read_field` and `yaml_read_list` in `common.sh`.
- **JSON handling**: NEVER use `jq`. Bash uses `awk`/`sed` template-based string building; PowerShell uses `ConvertTo-Json`/`ConvertFrom-Json`.
- **Color output**: `$'\033[...]'` ANSI codes in `common.sh`, disabled when stdout is not a TTY.
- **Error handling**: `set -euo pipefail` in all Bash scripts. `$ErrorActionPreference = "Stop"` in PowerShell.
- **Prompts**: `prompt_yn` / `Confirm-Prompt` ‚Äî auto-accepts when `SKILL_YES=1` is set.
- **Testing**: TDD with `assert_eq` pattern in `tests/test-*.sh`.

## Install System

Items install into agent-specific directories under the target project (or global path when `--global`). Paths vary by agent ‚Äî check `bin/lib/agents.sh` for the full map.

**Deduplication**: when both `claude-code` and `github-copilot` are selected, the CLI installs only to `.claude/skills/<name>/` since Copilot also reads from that directory. If `copilot` is selected alone, it goes to `.github/skills/<name>/`.

**Lock file**: every install creates/updates `.skills-lock.json` in the target project. Teammates run `bin/skill install` (no args) to restore the exact same setup ‚Äî pinned versions, same agents.

**Profiles**: named bundles under `profiles/` install multiple items at once via `bin/skill install --profile <name>`.

**Guided setup**: copy [`SETUP-PROMPT.md`](SETUP-PROMPT.md) into any AI assistant for an interactive walkthrough (environment detection, skill selection, AGENTS.md setup). Use [`UNINSTALL-PROMPT.md`](UNINSTALL-PROMPT.md) for interactive removal.

## Skill Naming Convention

Skill and instruction directories use dot-separated namespacing to indicate their source:

| Prefix | Source | Example |
|--------|--------|---------|
| `obra.superpowers.*` | [obra/superpowers](https://github.com/obra/superpowers) | `obra.superpowers.brainstorming` |
| `mattpocock.skills.*` | [mattpocock/skills](https://github.com/mattpocock/skills) | `mattpocock.skills.diagnose` |
| `local.*` | Custom/project-specific skills | `local.context-sync` |

Format: `<owner>.<repo>.<skill-name>`. The `name` field in `manifest.yaml` MUST match the directory name.

When adding skills from a new upstream repo, ALWAYS use this pattern: `<github-owner>.<repo-name>.<skill-name>`.

## Content Format

Each skill/instruction has a `manifest.yaml`:

```yaml
name: obra.superpowers.my-skill
type: skill
description: Brief description.
tags: [tag1, tag2]
targets: [claude-code, github-copilot]
files: [SKILL.md]
version: "1.0.0"
source: https://github.com/obra/superpowers
```

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Must match the directory name (dot-separated namespace) |
| `type` | Yes | `skill`, `agent`, or `instruction` |
| `description` | Yes | One-line summary |
| `tags` | No | Searchable tags (include source identifier, e.g. `obra`, `mattpocock`) |
| `targets` | Yes | Which AI agents this item supports |
| `files` | Yes | List of files to install (relative to item dir) |
| `version` | No | Semver string (defaults to `0.0.0`) |
| `source` | No | GitHub URL of the upstream repo (omit for `local.*` items) |
